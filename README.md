# Resume Tailor - Automated ATS Optimizer

This tool automates tailoring your resume to specific job descriptions using Claude AI, maximizing ATS keyword scores without manual copy-paste.

## Quick Start

### 1. Prerequisites

- **Python 3.10+** with pip
- **Anthropic API key** from https://console.anthropic.com/
- MS Word or LibreOffice (for PDF conversion) - optional

### 2. Installation

```bash
pip install -r requirements.txt
```

### 3. Set Up API Key

**Linux/Mac:**
```bash
export ANTHROPIC_API_KEY="your-api-key-here"
```

**Windows CMD:**
```cmd
set ANTHROPIC_API_KEY=your-api-key-here
```

**PowerShell:**
```powershell
$env:ANTHROPIC_API_KEY="your-api-key-here"
```

### 4. Prepare Your Template

If you haven't already, run the ATS rebuild once:

```bash
python rebuild_cv.py
```

This creates `AamirAli_Resume_ATS.docx` from your original resume (tables → paragraphs, fonts → Calibri).

### 5. Two Ways to Use

#### Option A: Manual (Original 2-Step)

1. **Prep mode** - generate prompt
   ```bash
   python tailor_resume.py --mode prep --jd job_description.txt
   ```
   Copy the prompt from `prompt.txt` and paste into Claude chat (claude.ai)

2. **Apply mode** - after Claude replies, save as `response.json`
   ```bash
   python tailor_resume.py --mode apply
   ```

#### Option B: Automated Batch (NEW - Recommended!)

**Fully automated** - no manual copy-paste:

1. Place job descriptions in `jd/` folder
   ```
   jd/
   ├── Titanbay_Senior_Analytics_Engineer.txt
   ├── Global_Lead_Data_Engineer.txt
   ├── Zilch_Senior_Analytics_Engineer.txt
   └── ...
   ```

2. Run batch processor:
   ```bash
   python batch_process.py
   ```

   That's it! It will:
   - Read each JD file
   - Send to Claude API directly
   - Save JSON responses to `responses/`
   - Generate tailored `.docx` and `.pdf` files in `generated_resumes/`
   - Create a summary report `batch_summary.md`

**Optional flags:**
```bash
python batch_process.py --model claude-haiku-3-5  # cheaper, faster
python batch_process.py --dry-run                # preview without API calls
python batch_process.py --skip-existing         # skip already processed JDs
```

## Filename Convention

For batch processing, name JD files as:

```
CompanyName_JobTitle.txt
```

Examples:
- `Titanbay_Senior_Analytics_Engineer.txt`
- `Global_Lead_Data_Engineer_MLOps.txt`
- `Zilch_Senior_Analytics_Engineer.txt`

**Spaces are OK too:** `Company Name - Job Title.txt`

The script extracts company and role from the filename to label outputs.

## What Gets Tailored

The tool updates these sections in your resume:

- **Skills** - adds missing keywords from JD (from predefined adjacent skills)
- **Experience bullets** - rewrites each bullet to include JD keywords (~200 chars, STAR format)
- **ATS compliance** - checks for tables, text boxes, multi-column layouts

All formatting (bold markers, bullet characters) is preserved.

## Tips for Best Results

1. **Quality JDs matter** - paste the full job description, not just bullet points
2. **Check ATS score** - The JSON response includes `ats_match_percentage`. Aim for 80%+
3. **Review before applying** - Open the generated docx and verify it looks good
4. **Iterate** - If ATS score is low, consider:
   - Adding missing keywords to your CV_CONTENT permanently
   - Re-running with a stronger model (Sonnet vs Haiku)
   - Manually tweaking bullet points after generation

## Troubleshooting

**"ANTHROPIC_API_KEY not set"**
- Set the environment variable as shown above
- Verify with: `echo $ANTHROPIC_API_KEY` (Mac/Linux) or `echo %ANTHROPIC_API_KEY%` (Windows)

**"Template not found: AamirAli_Resume_ATS.docx"**
- Run `python rebuild_cv.py` first to create it
- Or create a symlink to your existing template

**PDF conversion fails**
- Install MS Word (docx2pdf) or LibreOffice
- Or use online converters: upload .docx to Google Docs → Download as PDF
- Set `PDF_CONVERSION = "skip"` in code if you don't need PDFs

**JSON parse errors**
- Claude sometimes adds markdown fences - the script strips them automatically
- If still failing, check `debug_claude_response.txt` for raw output
- Try a different model: `--model claude-sonnet-4-6`

**Rate limits**
- The script adds 2-second delays between batch calls
- If you hit limits, increase the delay or upgrade your Anthropic plan

## Git Workflow

This repo is set up to exclude:
- Generated resumes (`*.docx`, `generated_resumes/`)
- API responses (`responses/`)
- Temporary files (`*.log`, `__pycache__/`, `.env`)

**Tracked files** (safe to commit):
- `tailor_resume.py`
- `batch_process.py`
- `rebuild_cv.py`
- `requirements.txt`
- `jd/` folder (your job descriptions)
- `.gitignore`

**Never commit:**
- `AamirAli_Resume_ATS.docx` (contains personal info)
- API responses (may contain PII from JDs)
- `ANTHROPIC_API_KEY` or any `.env` files

## Cost Estimate

**Claude pricing (as of 2025):**
- Sonnet 4.6: ~$3 per 1M input tokens, $15 per 1M output
- Haiku 3.5: ~$0.25 per 1M input, $1.25 per 1M output

**Typical batch:** 1 JD = ~2k input tokens (your resume + JD) + ~1k output
- Haiku: ~$0.001 per JD
- Sonnet: ~$0.015 per JD

So 100 job applications = ~$1 (Haiku) to ~$15 (Sonnet).
