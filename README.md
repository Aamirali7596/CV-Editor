# Resume Tailor - ATS Optimizer

This tool helps you tailor your resume to specific job descriptions using Claude AI, maximizing ATS keyword matching through a simple two-step workflow.

## Quick Start

### 1. Prerequisites

- **Python 3.10+** with pip
- **Anthropic API key** from https://console.anthropic.com/
- MS Word or LibreOffice (for PDF conversion) - optional

### 2. Installation

```bash
pip install -r requirements.txt
```

### 3. Set Up API Key

**Option A: .env file (Easiest)**
```powershell
# Copy the template:
Copy-Item .env.example .env
```
Then edit `.env` and paste your API key.

**Option B: Environment variable**
```bash
# Linux/Mac:
export ANTHROPIC_API_KEY="your-api-key-here"

# Windows CMD:
set ANTHROPIC_API_KEY=your-api-key-here

# PowerShell:
$env:ANTHROPIC_API_KEY="your-api-key-here"
```

**Option C: Permanent (System Environment Variable)**
- Press `Win + X` → System → Advanced system settings
- Environment Variables → New User Variable
- Name: `ANTHROPIC_API_KEY`, Value: `your-key`
- Restart terminal

### 4. Prepare Your Template

If you haven't already, run the ATS rebuild once:

```bash
python rebuild_cv.py
```

This creates `AamirAli_Resume_ATS.docx` from your original resume (tables → paragraphs, fonts → Calibri).

### 4. Two-Step Workflow

#### Step 1: Generate Prompt

Generate a tailored prompt from a job description:

```bash
python tailor_resume.py --mode prep --jd job_description.txt
```

This creates `prompt.txt` with a prompt optimized for your resume. Copy its contents and paste it into Claude's chat interface at [claude.ai](https://claude.ai).

Alternatively, you can manually paste the JD when prompted:
```bash
python tailor_resume.py --mode prep
```

#### Step 2: Apply Claude's Response

After Claude responds with the JSON, save the response as `response.json` in the project folder:

```bash
python tailor_resume.py --mode apply
```

This generates:
- A tailored `.docx` file in `generated_resumes/`
- A PDF version (if MS Word or LibreOffice is available)

Check the ATS compliance of your generated resume:

```bash
python tailor_resume.py --mode check --doc generated_resumes/Your_Company.docx
```

## What Gets Tailored

The tool updates these sections in your resume:

- **Skills** - adds missing keywords from JD (from predefined adjacent skills)
- **Experience bullets** - rewrites each bullet to include JD keywords (~200 chars, STAR format)
- **ATS compliance** - checks for tables, text boxes, multi-column layouts

All formatting (bold markers, bullet characters) is preserved.

## Tips for Best Results

1. **Quality JDs matter** - paste the full job description, not just bullet points
2. **Check ATS score** - The JSON response includes `ats_match_percentage`. Aim for 80%+
3. **Review before applying** - Open the generated docx and verify it looks good
4. **Iterate** - If ATS score is low, consider:
   - Adding missing keywords to your CV_CONTENT permanently
   - Re-running with a stronger model (Sonnet vs Haiku)
   - Manually tweaking bullet points after generation

## Troubleshooting

**"ANTHROPIC_API_KEY not found"**
- Create `.env` file and add your key (see `.env.example`)
- Or set environment variable: `echo $ANTHROPIC_API_KEY` to verify it's set
- Make sure you're in the project directory when running the script

**"Template not found: AamirAli_Resume_ATS.docx"**
- Run `python rebuild_cv.py` first to create it from your original resume
- Or create a symlink to your existing template

**PDF conversion fails**
- Install MS Word (docx2pdf) or LibreOffice
- Or use online converters: upload .docx to Google Docs → Download as PDF
- The script will still generate the .docx even if PDF fails

**JSON parse errors**
- Make sure you're copying Claude's full JSON response (no extra text)
- Claude sometimes adds markdown fences - the script strips them automatically
- If errors persist, check the raw response in `debug_claude_response.txt`

**Rate limits**
- Anthropic has rate limits based on your plan
- If you hit limits, wait a minute and retry or upgrade your Anthropic plan

## Advanced Usage

### Custom prompt engineering
Edit `build_prompt()` in `tailor_resume.py` to change how prompts are constructed (temperature, system prompt, JSON schema, etc.).

### Update skills globally
After adding new skills to `CV_CONTENT` in `tailor_resume.py`, you should also update the skills section in `AamirAli_Resume_ATS.docx` manually to keep them in sync.

### Batch processing (manual)
If you want to process multiple JDs without fully automated batch, you can script it:

```bash
for jd in jd/*.txt; do
    python tailor_resume.py --mode prep --jd "$jd"
    # Manually copy prompt.txt to Claude, save response as response.json
    # Then run: python tailor_resume.py --mode apply
done
```

## Git Workflow

This repo is set up to exclude:
- Generated resumes (`*.docx`, `generated_resumes/`)
- Temporary files (`prompt.txt`, `response.json`, `*.log`, `__pycache__/`, `.env`)

**Tracked files** (safe to commit):
- `tailor_resume.py`
- `rebuild_cv.py`
- `requirements.txt`
- `jd/` folder (your job descriptions)
- `.gitignore`

**Never commit:**
- `AamirAli_Resume_ATS.docx` (contains personal info)
- `response.json` (may contain PII from JDs)
- `ANTHROPIC_API_KEY` or any `.env` files
